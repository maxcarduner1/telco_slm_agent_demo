2. SCOPE AND APPLICABILITY

2.1 Functional Scope
This specification defines the Open Fronthaul interface between O-RAN Distributed Unit (O-DU) and O-RAN Radio Unit (O-RU) based on functional split 7.2x architecture. The specification covers Control Plane (C-Plane), User Plane (U-Plane), Synchronization Plane (S-Plane), and Management Plane (M-Plane) protocols and procedures.

2.2 Network Applicability
- 4G LTE networks with O-RAN architecture
- 5G NR networks (FR1 and FR2)
- Private networks and enterprise deployments
- Multi-vendor RAN deployments
- Cloud RAN (C-RAN) implementations

2.3 Deployment Scenarios
- Indoor small cells
- Outdoor macro cells
- Massive MIMO configurations
- Distributed antenna systems
- Mobile fronthaul networks

3. KEY DEFINITIONS AND ACRONYMS

3.1 Primary Definitions

O-DU (O-RAN Distributed Unit): Network function that implements RLC, MAC, and high-PHY layer functionality

O-RU (O-RAN Radio Unit): Network function that implements low-PHY layer functionality and RF processing

Fronthaul: Transport network connecting O-DU and O-RU functions

Split 7.2x: Functional split between high-PHY and low-PHY with enhanced control plane features

3.2 Key Acronyms

eCPRI: Enhanced Common Public Radio Interface
IQ: In-phase and Quadrature
PRB: Physical Resource Block
SCS: Subcarrier Spacing
PRACH: Physical Random Access Channel
PUSCH: Physical Uplink Shared Channel
PDSCH: Physical Downlink Shared Channel
BFW: Beamforming Weights
FFT: Fast Fourier Transform
CP: Cyclic Prefix
OFDM: Orthogonal Frequency Division Multiplexing

4. TECHNICAL SPECIFICATIONS AND PARAMETERS

4.1 Transport Layer Specifications

4.1.1 eCPRI Transport Parameters
- eCPRI Protocol Version: 2.0
- Maximum Transmission Unit: 9600 bytes
- Ethernet Frame Size: 64 to 9600 bytes
- VLAN Support: IEEE 802.1Q compliant
- Quality of Service: IEEE 802.1p priority marking

4.1.2 Network Layer Requirements
- IPv4/IPv6 dual stack support
- UDP transport protocol
- Maximum packet size: 9000 bytes (jumbo frames)
- Minimum bandwidth: 10 Gbps per O-RU connection
- Maximum latency: 250 microseconds one-way

4.2 Synchronization Requirements

4.2.1 Time Synchronization
- Precision Time Protocol: IEEE 1588v2 (PTP)
- Clock accuracy: ±1.5 microseconds
- Frequency accuracy: ±0.002 ppm
- Phase accuracy: ±1.3 microseconds
- Synchronization message rate: 128 packets per second

4.2.2 Frame Timing Parameters
- Radio Frame Duration: 10 milliseconds
- Slot Duration (15 kHz SCS): 1 millisecond
- Slot Duration (30 kHz SCS): 0.5 milliseconds
- Symbol Duration (15 kHz SCS): 71.43 microseconds
- Cyclic Prefix Duration: 4.69 microseconds (normal CP)

4.3 Control Plane Message Specifications

4.3.1 Section Type Parameters
- Section Type 0: Unused Resource Blocks
- Section Type 1: Most DL/UL Radio Channels
- Section Type 3: PRACH and Mixed-numerology Channels
- Section Type 5: UE-specific Beamforming Weights
- Section Type 6: Channel Information
- Section Type 7: LAA (License Assisted Access)

4.3.2 Compression Parameters
- Block Floating Point: 8, 9, 10, 12, 14, 16 bits
- Block Scaling: Dynamic scaling per PRB
- Mu-Law Compression: 8-bit compressed samples
- Modulation Compression: QPSK, 16QAM, 64QAM, 256QAM support

4.4 User Plane Data Specifications

4.4.1 IQ Data Parameters
- Sample Resolution: 8 to 16 bits per I/Q component
- Sample Rate: 30.72 MHz (20 MHz channel)
- Sample Rate: 61.44 MHz (40 MHz channel)
- Sample Rate: 122.88 MHz (80 MHz channel)
- Sample Rate: 245.76 MHz (160 MHz channel)

4.4.2 Resource Block Mapping
- PRB Size: 12 subcarriers
- Maximum PRBs (20 MHz): 100 PRBs
- Maximum PRBs (100 MHz): 273 PRBs
- Subcarrier Spacing: 15, 30, 60, 120 kHz
- FFT Size: 256, 512, 1024, 2048, 4096 points

5. PROCEDURES AND MESSAGE FLOWS

5.1 O-RU Startup Procedure

Step 1: Physical Layer Initialization
O-RU performs hardware initialization and self-test procedures

Step 2: Transport Layer Establishment
O-RU establishes Ethernet connectivity and obtains IP configuration

Step 3: Synchronization Acquisition
O-RU synchronizes to PTP grandmaster clock within 30 seconds

Step 4: Management Plane Registration
O-RU registers with O-DU management function using NETCONF protocol

Step 5: Control Plane Initialization
O-DU sends initial configuration parameters via C-Plane messages

Step 6: User Plane Activation
Data flow commences after successful C-Plane parameter validation

5.2 Beamforming Configuration Flow

Step 1: Beam Index Assignment
O-DU assigns unique beam indices for each spatial layer

Step 2: Beamforming Weight Distribution
O-DU transmits BFW coefficients using Section Type 5 messages

Step 3: Weight Application Timing
O-RU applies beamforming weights at specified symbol boundaries

Step 4: Beam Measurement Reporting
O-RU reports beam quality metrics via M-Plane interface

5.3 PRACH Processing Procedure

Step 1: PRACH Configuration
O-DU configures PRACH parameters via Section Type 3 messages

Step 2: Preamble Detection
O-RU performs preamble detection and timing estimation

Step 3: Result Reporting
O-RU reports detected preambles with timing advance information

Step 4: Response Generation
O-DU generates Random Access Response based on detection results

6. IMPLEMENTATION NOTES FOR NETWORK OPERATORS

6.1 Network Planning Considerations

6.1.1 Fronthaul Capacity Planning
Calculate required fronthaul bandwidth using formula:
Bandwidth = (Sample Rate × Sample Width × 2 × Compression Factor × Number of Antenna Ports)

Example for 4T4R 20MHz configuration:
Bandwidth = 30.72 MHz × 16 bits × 2 × 0.75 × 4 = 2.95 Gbps

6.1.2 Latency Budget Allocation
Total one-way latency budget: 250 microseconds
- Network transport delay: 150 microseconds maximum
- O-DU processing delay: 50 microseconds maximum
- O-RU processing delay: 50 microseconds maximum

6.2 Deployment Guidelines

6.2.1 Synchronization Network Design
Deploy IEEE 1588v2 grandmaster clocks with GPS/GNSS reference
Implement boundary clocks at network aggregation points
Configure PTP domains to isolate synchronization traffic

6.2.2 Quality of Service Configuration
Map C-Plane traffic to highest priority queue (Priority 7)
Map U-Plane traffic to high priority queue (Priority 6)
Map M-Plane traffic to medium priority queue (Priority 4)
Configure traffic shaping to prevent buffer overflow

6.3 Performance Optimization

6.3.1 Compression Algorithm Selection
Use Block Floating Point for channels requiring high dynamic range
Apply Mu-Law compression for voice-centric applications
Implement adaptive compression based on channel conditions

6.3.2 Buffer Management
Configure O-RU receive buffers for 4-symbol advance reception
Set O-DU transmission timing to account for network jitter
Implement adaptive buffer sizing based on measured latency variation

6.4 Troubleshooting Guidelines

6.4.1 Common Issues
- Synchronization loss: Verify PTP configuration and network path
- High latency: Check network congestion and QoS configuration  
- IQ data corruption: Validate compression parameters and bit alignment
- Beamforming errors: Verify weight coefficient precision and timing

6.4.2 Monitoring Parameters
Monitor PTP offset and frequency accuracy continuously
Track fronthaul utilization and packet loss statistics
Measure C-Plane message processing delays
Validate U-Plane data integrity using built-in checksums
