2. SCOPE AND APPLICABILITY

2.1 General Scope
This specification defines the Radio Resource Control (RRC) protocol for the radio interface between User Equipment (UE) and the Next Generation NodeB (gNB) in 5G New Radio (NR) systems. The protocol operates at Layer 3 of the radio protocol stack.

2.2 Functional Coverage
- RRC connection establishment and release procedures
- System information broadcasting and acquisition
- Measurement configuration and reporting mechanisms
- Mobility management including handover procedures
- RRC state management and transitions
- Security activation and key management
- Radio bearer establishment and configuration

2.3 Network Applicability
- 5G Standalone (SA) deployments
- 5G Non-Standalone (NSA) deployments with LTE anchor
- Private network implementations
- Public network operator deployments

3. KEY DEFINITIONS AND ACRONYMS

3.1 Primary Definitions

RRC Connection: Logical connection between UE and gNB for dedicated control signaling

RRC State: Operating mode of UE RRC entity determining available procedures and power consumption characteristics

Measurement Object: Set of cells on which UE performs measurements as configured by the network

Reporting Configuration: Set of criteria triggering measurement reports from UE to network

4. TECHNICAL SPECIFICATIONS AND PARAMETERS

4.1 RRC State Machine Parameters

RRC_IDLE State:
- No RRC connection established
- UE performs cell selection/reselection
- Paging monitoring cycle: 32, 64, 128, 256 frames
- Battery saving timer T320: 5min, 10min, 20min, 30min, 60min, 120min, 180min

RRC_INACTIVE State:
- RRC connection suspended
- UE context maintained in gNB and AMF
- RNA update timer T380: 5min, 10min, 20min, 30min, 60min, 120min, 180min
- Periodic RNA update timer T320: 5min, 10min, 20min, 30min, 60min, 120min, 180min

RRC_CONNECTED State:
- Active RRC connection
- Dedicated signaling and data transfer
- UE-specific configuration active
- Network-controlled mobility

4.2 Measurement Configuration Parameters

4.2.1 RSRP Measurement Range
- Measurement range: -156 dBm to -31 dBm
- Measurement resolution: 1 dB
- Reporting range: 0 to 127 (mapped values)

4.2.2 RSRQ Measurement Range  
- Measurement range: -43 dB to 20 dB
- Measurement resolution: 0.5 dB
- Reporting range: 0 to 127 (mapped values)

4.2.3 Measurement Timing Parameters
- Layer 3 filtering coefficient a: fc0, fc1, fc2, fc3, fc4, fc5, fc6, fc7, fc8, fc9, fc11, fc13, fc15, fc17, fc19
- Measurement period for serving cell: 200ms (default)
- Measurement period for neighbor cells: 200ms to 2560ms
- Time to trigger values: 0ms, 40ms, 64ms, 80ms, 100ms, 128ms, 160ms, 256ms, 320ms, 480ms, 512ms, 640ms, 1024ms, 1280ms, 2560ms, 5120ms

4.3 Handover Parameters

4.3.1 Event-Based Handover Thresholds
- Event A1 (Serving better than threshold): -156 dBm to -31 dBm (RSRP), -43 dB to 20 dB (RSRQ)
- Event A2 (Serving worse than threshold): Same range as A1
- Event A3 (Neighbor better than serving plus offset): Offset range -15 dB to 15 dB
- Event A4 (Neighbor better than threshold): Same range as A1
- Event A5 (Serving worse than threshold1 AND neighbor better than threshold2): Dual threshold configuration

4.3.2 Handover Execution Timers
- T304 (Handover timer): 50ms, 100ms, 150ms, 200ms, 500ms, 1000ms, 2000ms, 10000ms
- T310 (Radio link failure timer): 0ms, 50ms, 100ms, 200ms, 500ms, 1000ms, 2000ms
- T311 (RRC re-establishment timer): 1000ms, 3000ms, 5000ms, 10000ms, 15000ms, 20000ms, 30000ms

5. RELEVANT PROCEDURES AND MESSAGE FLOWS

5.1 RRC Connection Establishment Procedure

Step 1: UE initiates procedure by sending RRCSetupRequest on CCCH
   - Message contains: UE identity, establishment cause, spare bits
   - Establishment causes: emergency, highPriorityAccess, mt-Access, mo-Signalling, mo-Data, mo-VoiceCall, mo-VideoCall, mo-SMS, mps-PriorityAccess, mcs-PriorityAccess

Step 2: gNB responds with RRCSetup message on CCCH
   - Contains: RRC transaction identifier, radio bearer configuration, master cell group configuration

Step 3: UE completes establishment with RRCSetupComplete on DCCH
   - Contains: Selected PLMN identity, registered AMF, S-TMSI, dedicated NAS message

Step 4: gNB may send RRCReconfiguration for additional configuration
   - Contains: Radio bearer configuration, measurement configuration, mobility control information

5.2 Measurement Reporting Procedure

Step 1: gNB configures measurements via RRCReconfiguration message
   - Measurement objects: serving cell, neighbor cell lists
   - Reporting configurations: event criteria, periodic reporting
   - Measurement identities: linking objects to reporting configs

Step 2: UE performs measurements according to configuration
   - Layer 1 measurements: RSRP, RSRQ, SINR
   - Layer 3 filtering applied using configured coefficient
   - Event evaluation performed every measurement period

Step 3: UE sends MeasurementReport when criteria met
   - Contains: Measurement identity, measured results, additional measurement information
   - Triggered by: Event-based criteria or periodic timer expiry

Step 4: gNB processes report and may initiate mobility procedure
   - Handover decision based on measurement results
   - Load balancing considerations
   - Quality of service requirements

5.3 Handover Execution Procedure

Step 1: Source gNB makes handover decision
   - Based on measurement reports and network policies
   - Considers target cell capacity and interference levels

Step 2: Source gNB sends Handover Request to target gNB via Xn interface
   - Contains: UE context, security context, bearer information

Step 3: Target gNB responds with Handover Request Acknowledge
   - Includes: Target cell configuration, new C-RNTI, security parameters

Step 4: Source gNB sends RRCReconfiguration with mobilityControlInfo
   - Contains: Target cell identity, new configuration, random access parameters
   - Timer T304 started for handover completion

Step 5: UE performs random access to target cell
   - Uses provided RACH configuration
   - Establishes uplink synchronization

Step 6: UE sends RRCReconfigurationComplete to target gNB
   - Confirms successful handover completion
   - Timer T304 stopped

6. IMPLEMENTATION NOTES FOR NETWORK OPERATORS

6.1 Measurement Configuration Optimization

6.1.1 Layer 3 Filtering Configuration
- Configure appropriate filtering coefficient based on mobility scenario
- High mobility: Use higher filtering (fc4-fc6) to reduce ping-pong effects
- Low mobility: Use lower filtering (fc0-fc2) for faster response to channel changes
- Indoor scenarios: Consider fc2-fc4 for balance between stability and responsiveness

6.1.2 Time-to-Trigger Optimization
- Dense urban areas: Use longer TTT (320ms-640ms) to avoid unnecessary handovers
- Highway scenarios: Use shorter TTT (80ms-160ms) for rapid handover execution
- Indoor-outdoor boundaries: Configure intermediate values (160ms-320ms)

6.2 Handover Parameter Tuning

6.2.1 A3 Offset Configuration
- Typical range: 2-6 dB for macro-macro handovers
- Macro-small cell: 0-3 dB to encourage offloading
- Small cell-macro: 3-8 dB to provide hysteresis
- Consider interference levels and cell overlap areas

6.2.2 Timer T304 Configuration
- Macro cells: 1000ms-2000ms depending on cell size
- Small cells: 500ms-1000ms for faster handover completion
- High-speed scenarios: Consider 2000ms to accommodate processing delays
- Monitor handover failure rates to optimize timer values
